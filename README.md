# yuangaogao.github.io

Source repository for [yuangaogao.github.io](https://yuangaogao.github.io/) — personal academic website for Yuan Gao.

Built with [Hugo](https://gohugo.io/) (v0.112.3) using the [researcher](https://github.com/ojroques/hugo-researcher) theme. Deployed automatically to GitHub Pages via GitHub Actions on every push to `master`.

---

## How Publishing Works

Pushing any commit to `master` automatically:
1. Triggers `.github/workflows/hugo.yml`
2. GitHub Actions runs `hugo --gc --minify`
3. The built `public/` is deployed to GitHub Pages

**No manual build step needed.** Just edit source files and push.

### Local Preview

```bash
# from repo root
hugo server
# then open http://localhost:1313
```

## Editing Content

| What to edit | File |
|---|---|
| Home page | `content/_index.md` |
| Publications | `content/pubs.md` |
| Research descriptions | `content/research.md` |
| Teaching | `content/teach.md` |
| Activities | `content/activity.md` |
| Site config, nav, colors | `config.toml` |
| Seminar schedule | `static/seminar.html` |

### Adding a new paper PDF

1. Drop the PDF into `content/papers/`
2. Add a link in `content/pubs.md` using the path `/papers/FILENAME.pdf`

### Updating the CV

1. Drop the new PDF into `static/` (e.g. `static/cv_gy_202601.pdf`)
2. Update the `url` under `[[menu.main]] name = "CV"` in `config.toml`

## Seminar Auto-Update

`.github/workflows/update-seminar.yml` runs daily at 12:00 UTC (7 AM ET).

It runs `scripts/update_seminar.py`, which fetches the seminar schedule from a Google Doc and patches `static/seminar.html`. If the file changes, the bot commits it, which triggers `hugo.yml` to redeploy.

To trigger manually: **Actions → Update Seminar → Run workflow**.
